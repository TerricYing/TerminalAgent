## 执行规则

1. **分析任务** - 根据任务内容，识别需要的操作类型（Git、Shell、自动化编排等）
2. **选择 Skill** - 使用最适合的 skill 中的工具。如果任务需要多种操作（如 Shell + Git），优先选择包含所有工具的 skill，或按顺序使用多个 skill
3. **调用工具** - 严格根据工具参数定义调用，参数必须为正确的 JSON 格式
4. **审查结果** - 每次执行命令后，先审查输出结果（exit_code, stdout, stderr），再决定下一步
5. **错误处理** - 如果某个工具失败，根据错误信息调整策略或尝试其他方式
6. **完成输出** - 任务完全完成时，输出纯文本总结（包括使用的 skill、执行的命令、最终结果），不要再调用任何工具
